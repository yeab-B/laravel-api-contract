# Glossary of Terms

Understanding the terminology used within the `laravel-api-contract` ecosystem is crucial for configuration and contribution. This glossary defines all the important terms used throughout the documentation and source code.

---

### Abstract Syntax Tree (AST)
A tree representation of the abstract syntactic structure of source code. The package's `ResourceAnalyzer` uses AST parsing to read the source code of your `JsonResource` classes and extract the exact array keys and values returned by the `toArray()` method without executing the code.

### Analyzer
A class responsible for extracting metadata from the host Laravel application. Analyzers (like `RouteAnalyzer` and `ControllerAnalyzer`) do not generate output; they solely read code and return standardized Data Transfer Objects (DTOs).

### API Contract (`ApiContract`)
The central, definitive intermediate representation of your entire API. It is a strictly typed JSON object (or PHP class in memory) that merges routes, validation rules, and response schemas into one single source of truth.

### Artifact
Any file generated by the package based on the API Contract. Artifacts include `swagger.json`, TypeScript interface files (`api.ts`), Postman collections, Markdown files, and PHPUnit test stubs.

### Baseline Contract
When comparing API versions, the baseline contract is the older, previously deployed contract (often passed via the `--old` flag). The comparator checks the new contract against the baseline to detect breaking changes.

### Breaking Change
A modification to the API that will cause existing, perfectly functioning client integrations (like a mobile app or frontend SPA) to crash or fail. Examples include removing a field from a response, changing a response type, or making an optional input required.

### Contract Builder
The orchestration engine (`ContractBuilder`) that iteratively passes routes to various analyzers and merges their outputs into the master `ApiContract` object.

### Data Transfer Object (DTO)
A simple object used to encapsulate data and send it from one subsystem of an application to another without behavior or business logic. `laravel-api-contract` uses DTOs heavily in its `Services\DTO` namespace (e.g., `EndpointDefinition`, `RequestDefinition`).

### Endpoint Definition
The unified DTO that represents a single API route. It merges the HTTP method, URI, required inputs (from Form Requests), and output schema (from Json Resources) into one object within the API Contract.

### Generator
A class responsible for consuming the `ApiContract` object and mapping its structure into a specific language or format. Generators (like `TypeScriptGenerator` and `SwaggerGenerator`) have no knowledge of Laravel; they only understand the Contract structure.

### Path Traversal
A security vulnerability where an attacker manipulates file paths (e.g., using `../../`) to access or overwrite directories outside the intended bounds. The package actively defends against this via the `Configuration::ensureSafePath()` mechanism.

### Reflection
A feature in PHP that allows a program to inspect its own classes, interfaces, properties, and methods at runtime. The package uses Reflection to read controller signatures and type-hints without using `eval()`.

### Semantic Versioning (SemVer)
A standard versioning system (`MAJOR.MINOR.PATCH`). In the context of API Contracts, a MAJOR version bump indicates a breaking change, a MINOR version bump indicates a backwards-compatible feature addition, and a PATCH indicates a bug fix.

### Single Source of Truth (SSOT)
The architectural concept that data should be defined in exactly one place. In this ecosystem, your standard Laravel PHP code is the single source of truth. All documentation and frontend types are derived from it, preventing synchronization drift.
